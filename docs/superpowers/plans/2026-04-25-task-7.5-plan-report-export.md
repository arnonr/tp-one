# Task 7.5: Plan Report Export — Implementation Plan

**Date:** 2026-04-25
**Status:** Pending Approval

## Requirements

จาก spec: PDF report with progress charts, Excel export with monthly data, Thai fiscal year quarter grouping

### สิ่งที่มีอยู่แล้ว
- `pdfkit` + `exceljs` installed ใน server dependencies
- Thai font files (Sarabun) ที่ `packages/server/src/assets/fonts/`
- Report module PDF/Excel services เป็น pattern ที่ใช้ได้เป็นตัวอย่าง
- Plan progress API (`getPlanProgress`) ส่งข้อมูล hierarchy 4 ระดับ
- ExcelJS audit log export ใน plan module ทำไว้แล้ว

### สิ่งที่ต้องสร้าง
1. **PDF export** — รายงานแผนปฏิบัติการพร้อม progress bars + charts
2. **Excel export** — ข้อมูลรายเดือน จัดกลุ่มตามไตรมาสงบประมาณ
3. **Frontend UI** — ปุ่ม export ในหน้า PlanDetailView

## Implementation Steps

### Step 1: Backend Types — Plan Export Types

**File:** `packages/server/src/modules/plan/types.ts`

เพิ่ม types สำหรับ export:
```typescript
interface PlanExportQuery {
  period?: ProgressPeriod  // monthly | quarterly | yearly
  format: 'pdf' | 'excel'
}
```

### Step 2: Backend Service — Plan PDF Export

**File:** `packages/server/src/modules/plan/plan-pdf.service.ts` (new file)

สร้าง PDF generation service:
- รับ `PlanProgress` data + plan metadata
- Reuse Sarabun fonts จาก report module
- Layout:
  - หัวกระดาษ: ชื่อแผน, ปีงบ, วันที่พิมพ์ (พ.ศ.)
  - Progress overview bar (overall %)
  - ตาราง Strategy → Goal → Indicator พร้อม progress bars
  - Thai fiscal year quarter grouping (Q1=ต.ค.-ธ.ค. ...)
- Export helpers: thai number, progress bar drawing, table rendering
- Return `Buffer`

### Step 3: Backend Service — Plan Excel Export

**File:** `packages/server/src/modules/plan/plan-excel.service.ts` (new file)

สร้าง Excel generation service:
- รับ `PlanProgress` data + plan metadata + monthly indicator updates
- Reuse ExcelJS pattern จาก audit log export
- Sheets:
  - **สรุปภาพรวม** — overview: plan name, year, overall progress, per-strategy summary
  - **รายงานตามไตรมาส** — quarterly grouping with Thai labels
    - Q1 (ต.ค.-ธ.ค.), Q2 (ม.ค.-มี.ค.), Q3 (เม.ย.-มิ.ย.), Q4 (ก.ค.-ก.ย.)
  - **รายละเอียดตัวชี้วัด** — full indicator detail with monthly values
- Styled headers, borders, progress % columns
- Thai headers, Buddhist year formatting
- Return `Buffer`

### Step 4: Backend Service — Data Aggregation

**File:** `packages/server/src/modules/plan/plan.service.ts`

เพิ่ม functions:
- `getPlanProgressForExport(planId: string, period: ProgressPeriod)` — เรียก `getPlanProgress` + enrich ด้วย plan metadata (name, year)
- `getMonthlyIndicatorUpdates(planId: string, fiscalYear: number)` — query indicator updates จัดกลุ่มตามเดือน เพื่อใช้ใน Excel monthly sheet
- `exportPlanPDF(planId: string, period: ProgressPeriod)` — orchestrate PDF generation
- `exportPlanExcel(planId: string, period: ProgressPeriod)` — orchestrate Excel generation

### Step 5: Backend Controller & Plugin — Export Endpoints

**File:** `packages/server/src/modules/plan/plan.controller.ts`

เพิ่ม handlers:
```typescript
async exportPlanPDF(user, params, query): Promise<Response>
async exportPlanExcel(user, params, query): Promise<Response>
```

**File:** `packages/server/src/modules/plan/plan.plugin.ts`

เพิ่ม routes:
```
GET /api/plans/:planId/export/pdf?period=monthly
GET /api/plans/:planId/export/excel?period=monthly
```

### Step 6: Frontend API — Export Calls

**File:** `packages/web/src/services/planApi.ts`

เพิ่ม functions:
```typescript
exportPlanPDF(planId: string, period?: string): Promise<void>
exportPlanExcel(planId: string, period?: string): Promise<void>
```
- Blob download pattern เดียวกับ audit log export

### Step 7: Frontend UI — Export Buttons

**File:** `packages/web/src/views/PlanDetailView.vue`

เพิ่ม export controls:
- Period selector dropdown (รายเดือน/รายไตรมาส/รายปี)
- Export PDF button
- Export Excel button
- Loading state ระหว่าง generate

## Data Flow

```
[User clicks Export PDF/Excel]
  → Frontend: planApi.exportPlanXXX(planId, period)
  → Backend: GET /api/plans/:planId/export/pdf?period=monthly
  → Controller: validates user + planId
  → Service: getPlanProgress() + getMonthlyUpdates()
  → PDF/Excel Service: generate Buffer with Thai formatting
  → Response: Content-Disposition attachment + Buffer
  → Frontend: Blob download trigger
```
