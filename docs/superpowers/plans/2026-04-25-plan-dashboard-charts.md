# Task 7.3: Plan Dashboard + Charts

## Requirements

1. **Progress chart per indicator** — monthly line chart showing indicator value vs target over time
2. **Category/Strategy progress bar** — visual progress bars for each strategy with weighted aggregation
3. **Plan overview with fiscal year filter** — dashboard showing all plans for a fiscal year with summary metrics and charts

## Architecture Decision

**Approach: Enhance AnnualPlanView as the plan dashboard**

Instead of a separate `/plans/dashboard` route, enhance the existing `/plans` (AnnualPlanView) to serve as both the plan list AND the plan dashboard. This keeps the navigation simple and matches how users think about it: "I go to แผนปฏิบัติการ and see my plans + overview charts."

**Route structure remains:**
- `/plans` → AnnualPlanView (dashboard + list)
- `/plans/:id` → PlanDetailView (single plan detail)

## Chart Details

### 1. Strategy Comparison Bar Chart (Horizontal)
- Type: ECharts bar (horizontal)
- X-axis: Progress percentage (0-100%)
- Y-axis: Strategy names (Thai)
- Color: Gradient based on progress level
- Data: Aggregated from `getPlanProgress()` per plan

### 2. Plan Status Doughnut
- Type: ECharts doughnut
- Segments: ร่าง (draft), กำลังดำเนินการ (active), เสร็จสิ้น (completed)
- Center text: Total plan count
- Colors: Grey, Blue, Green

### 3. Indicator Achievement Stacked Bar
- Type: ECharts stacked bar
- Groups: Per plan
- Stacks: <25%, 25-50%, 50-75%, 75-100% indicator counts
- Colors: Red, Orange, Blue, Green

### 4. Overall Progress Gauge (optional, if time permits)
- Type: ECharts gauge
- Shows average progress across all active plans
- Center label: percentage
